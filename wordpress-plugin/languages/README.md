# Translations

`orca-dam-picker.pot` is the extracted source template. Translators copy it to
`orca-dam-picker-{locale}.po` (e.g. `orca-dam-picker-nl_NL.po`) and fill in the
`msgstr` lines.

## Regenerating the .pot file

After editing PHP/JS strings (`__()`, `_e()`, `_x()`):

```bash
wp i18n make-pot . languages/orca-dam-picker.pot \
    --domain=orca-dam-picker \
    --exclude=node_modules,vendor,assets/build,tests
```

## Compiling .po → .mo (required for PHP runtime)

```bash
msgfmt languages/orca-dam-picker-nl_NL.po -o languages/orca-dam-picker-nl_NL.mo
```

## Splitting translations for the JS bundles

WordPress loads JS translations from per-handle `.json` files generated by:

```bash
wp i18n make-json languages/ --no-purge
```

This produces files like `orca-dam-picker-nl_NL-orca-dam-settings.json`, which
`wp_set_script_translations()` will load automatically.

The release CI workflow runs all three steps before zipping.
